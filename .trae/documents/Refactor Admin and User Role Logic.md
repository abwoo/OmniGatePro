# 重构管理员与普通用户权限逻辑

## 1. 后端逻辑重构 (Security & RBAC)
- **环境变量化管理**：在 `core/config.py` 中增加 `SUPER_ADMIN_EMAIL` 配置。
- **登录逻辑优化**：在 `api/main.py` 中，登录时检查邮箱。如果是 `SUPER_ADMIN_EMAIL`，则确保其数据库角色为 `ADMIN`。
- **注册逻辑锁死**：严禁通过公开注册接口创建任何 `ADMIN` 角色账号。

## 2. 前端 UI 与路由优化 (UX)
- **角色感知菜单**：修改 `App.tsx`，根据 API 返回的 `user.role` 动态显示/隐藏管理入口。
- **登录页清理**：移除登录页上的“API Config”默认展示（改为隐藏触发或仅开发模式可见），保持界面纯净。

## 3. 文档与安全清理
- **README 修正**：移除所有明文密码。增加“管理员配置”说明，指导您如何在本地 `.env` 中设置自己的专属管理账号。
- **环境隔离**：确保生产环境不包含任何默认测试账号。
