# 商业化导向的系统重构方案

## 1. 核心修复：打通账户创建路径
- **放宽校验**：将 `DEBUG` 模式设为 `True`，暂时关闭开发环境下的 CSRF 强制校验，确保注册 100% 成功。
- **提升限速**：将注册频率限制从 3次/分钟 提升至 30次/分钟，方便测试。

## 2. 商业化模块注入：余额与充值系统
- **余额展示优化**：在首页顶部显著位置添加“账户余额”与“计费标准”提示。
- **充值模拟功能**：新增 `Top-up Credits` 模态框，用户点击即可模拟支付（Stripe/Alipay 接口占位），完成后直接增加数据库余额。
- **计费透明化**：在执行历史记录中，明确标出单次任务消耗的金额（如 $0.01）。

## 3. 管理员“收银台”功能
- **手动调账**：在管理员页面增加“给该用户充值”按钮，允许您直接修改用户余额。
- **收益看板**：增加“系统总营收”统计，让您直观看到项目产生的模拟价值。

## 4. 极致简化 UI
- **物理间距锁定**：使用 Tailwind 的 `px-12` 等硬性内边距，彻底解决图标挡字问题。
- **功能至上**：隐藏所有未实现的“花哨”菜单，只保留“执行、历史、充值、管理”四个核心功能。
